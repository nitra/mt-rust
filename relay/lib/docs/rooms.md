## Огляд

Relay тримає кімнати за кореневим вузлом задачі, щоб розсилати `Envelope` усім підписникам і дати новому підписнику короткий live-реплей із уже наявного ефемерного буфера останніх N `Envelope`. Межа relay тут така: він не парсить payload далі роутінгових полів і не зберігає журнали сесій. Глибший реплей лишається поза relay і добирається хостом із `session.jsonl` за run ref-а. Публічний API: `BUFFER_LIMIT` і `Rooms`.

## Поведінка

- `BUFFER_LIMIT` — задає верхню межу кількості `Envelope`, які relay тримає в кімнаті для короткого live-реплею під час підписки.
- `Rooms` — веде кімнати за кореневим вузлом задачі, розсилає `Envelope` усім підписникам і віддає новому підписнику вже наявний буфер; не розбирає payload далі роутінгових полів і не зберігає журнали сесій, тому глибший реплей береться з `session.jsonl` через хост.

## Публічний API

- BUFFER_LIMIT — обмежує буфер кімнати до 200 Envelope за один run.
- Rooms — зберігає кімнати з тимчасовим буфером і списком підписників.
