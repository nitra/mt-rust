## Огляд

Файл надає read-only допоміжні засоби для тестових фікстур: `withTmpDir`, `leafDoc`, `generatedDoc` і `memoryReader`. Він потрібен, щоб тести відтворювано отримували ізольований тимчасовий контекст, готові тексти leaf- і generated-документів із маркерами повідомлень (test.mdc) та in-memory читання без запису у ФС чи БД. Помилки перехоплюються fail-safe і не виходять назовні.

## Поведінка

- `withTmpDir` створює ізольовану тимчасову теку для тестової фікстури за каноном `` і гарантовано прибирає її після виконання.
- `leafDoc` формує текст leaf-документа для тестового полігона з опційною суттю, деталями та draft-маркером.
- `generatedDoc` формує згенерований документ верхнього шару з актуальними контрольними мітками джерел.
- `memoryReader` надає in-memory читач файлів поверх переданого набору текстів і повертає порожній результат для відсутніх шляхів.

## Публічний API

- withTmpDir — створює ізольовану тимчасову теку для фікстур, прибирає її після виконання й не змінює робочу теку процесу.
- leafDoc — описує leaf-документацію полігона.
- generatedDoc — представляє згенеровану документацію верхнього шару з актуальними CRC вихідних файлів.
- memoryReader — читає вміст файлів із Map у памʼяті; підтримує сценарії з маркерами повідомлень (test.mdc).

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
