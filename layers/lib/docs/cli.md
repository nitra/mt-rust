## Огляд

Цей файл є точкою входу та оркестратором основної логіки програми. Він ініціалізує роботу, зчитуючи конфігурацію з `flags.json`, парсячи аргументи CLI за допомогою `parseArgv` та запускаючи основні модулі через `main`. Компоненти взаємодіють з файловою системою, а об'єкти IO створюються через `createIo`. Система забезпечує стійкість шляхом перехоплення помилок (`fail-safe`), не генеруючи винятків назовні, і повертає порожнє значення (`null`) у випадках невдачі, зокрема при роботі з файлами.

## Поведінка

Поведінка:
parseArgv розбирає передані аргументи командного рядка, визначаючи команду, шлях до документації та додаткові прапорці.
createIo створює обгортку для операцій з файловою системою, що працює у межах вказаного кореня документації, і при відсутності файлу повертає null.
main виконує основну логіку програми: зчитує конфігурацію, парсить аргументи та ініціює роботу відповідно до вказаної команди.

## Публічний API

parseArgv — зчитує та обробляє аргументи командного рядка.
createIo — надає механізм доступу до файлової системи, обмежений каталогом документації.
main — ініціалізує основний потік виконання програми.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
