## Огляд

Файл відповідає за облік та оновлення цілісності контенту файлів. Він забезпечує актуальність інформації для формування документів шляхом обчислення та оновлення CRC. Функція `currentCrcs` надає поточні значення CRC, тоді як `refreshFileCrcs` оновлює ці значення для файлів. `runBuild` та `bootstrapEssences` використовують цю логіку цілісності для коректного виконання збірки та ініціалізації есенцій.

## Поведінка

Поведінка:
currentCrcs обчислює пари CRC для вмісту файлу, визначаючи цілісність його основної частини та суті.
refreshFileCrcs виконує детерміноване оновлення `fileCrc` для документів, чия суть залишається незмінною.
runBuild генерує документ відповідного шару, використовуючи агреговані суті з нижчих шарів, або пропускає його, якщо стан документа не змінився.
bootstrapEssences автоматично створює чернетку секції "Суть" для листових документів, де ця секція відсутня.

## Публічний API

currentCrcs — фіксує актуальні криптографічні хеші (CRC) для кожного файлу-джерела.
refreshFileCrcs — оновлює збережений CRC для джерел, які зберігаються в режимі "деталі тільки", підтверджуючи, що основна суть документа залишається вірною без участі LLM.
runBuild — створює шари документації від нижчих до вищих рівнів; перед обробкою кожного джерела зчитує його CRC, щоб гарантувати, що новіші шари використовують актуальні дані попередніх.
bootstrapEssences — генерує початковий варіант секції «## Суть» для найменших довідників, які ще не мають такого опису; це маркер чернетки, який має підтвердити людина.
