## Огляд

Файл формує read-only статус актуальності відносно поточного вмісту джерел. Він потрібен, щоб обчислити стан через `computeStatus`, подати його для відображення через `renderStatus` і визначити код завершення через `statusExitCode`.

## Поведінка

- `computeStatus` визначає актуальність шарів документації та перекладів, порівнює записаний стан із поточним вмістом джерел і повертає зведений звіт без запису у ФС чи БД.
- `renderStatus` перетворює звіт стану на людиночитний або JSON-вивід і додає код завершення для CLI-сценаріїв.
- `statusExitCode` визначає код завершення за найгіршим станом документації, наявністю проблемних перекладів і режимом strict.

## Публічний API

- computeStatus — визначає стан шарів і перекладів на основі переданого читача файлів, без звернень до мережі чи файлової системи.
- renderStatus — формує зрозумілий для людини звіт про стан і відповідний код завершення.
- statusExitCode — повертає код завершення: успіх для актуального стану, помилку для застарілих чи чорнових даних, окрему помилку для відсутньої сутності.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
