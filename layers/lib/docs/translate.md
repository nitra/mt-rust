## Огляд

Файл готує Markdown-документи до багатомовного перекладу: `splitBySections` ділить великі тексти на послідовні частини, `rewriteLinks` адаптує локальні посилання для мовних версій, а `runTranslate` керує створенням перекладів. Він потрібен, щоб перекладати документи частинами та не змінювати зовнішні посилання.

## Поведінка

- `splitBySections` ділить великий Markdown-документ на послідовні фрагменти за межами H2-секцій, щоб перекладати обсяг, прийнятний для LLM, без зміни порядку вмісту.

- `rewriteLinks` замінює відносні посилання на `.md`-документи мовними версіями лише тоді, коли ціль входить до області перекладу; зовнішні URL і посилання поза областю лишаються без змін.

- `runTranslate` проходить документами області перекладу й мовами, пропускає актуальні переклади, у dry-run лише показує кандидатів, а для застарілих або відсутніх перекладів генерує новий Markdown із метаданими та підсумовує записані, пропущені й невдалі файли.

## Публічний API

- splitBySections — розбиває вміст за межами H2-секцій на частини в межах ліміту та збирає їх у стабільному порядку.
- rewriteLinks — замінює відносні посилання на `.md`-документи посиланнями на мовні версії, коли ціль входить до області перекладу.
- runTranslate — оновлює derived-переклади документації за змінами базових файлів, зберігає повну деталізацію оригіналу й не чіпає authored-переклади без зміни base.
