## Огляд

Цей файл налаштовує інструментарій для проведення мутаційного тестування, використовуючи `vitest` та спираючись на конфігураційні файли `mutation.json` та `incremental.json`. Він забезпечує швидкий аналіз, запускаючи лише тести, які покривають мутовані рядки. Результати зберігаються у форматах `json` та `clear-text`, що дозволяє відновлювати сеанс мутаційного тестування.

## Поведінка

1. Визначає, що для виконання тестів використовується `vitest` з конфігураційного файлу `vitest.config.mjs`.
2. Встановлює режим аналізу покриття як `perTest`, що забезпечує запуск лише тих тестів, які покривають мутовану лінію, підвищуючи швидкість.
3. Встановлює тимчасову директорію для звітів мутацій як `reports/stryker/.tmp`.
4. Визначає, що результати мутацій збираються у двох форматах: `json` та `clear-text`.
5. Вказує на збереження детального JSON-звіту про мутації у файл `reports/stryker/mutation.json`.
6. Вмикає інкрементальний режим, що дозволяє відновити результати після збоїв процесу.
7. Забезпечує збереження даних між запусками в файл `reports/stryker/incremental.json`, на який посилається інкрементальний режим.
